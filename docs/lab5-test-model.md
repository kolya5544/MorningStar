# Лабораторная работа №5. Тестовая модель MorningStar

## 1. Критические пользовательские сценарии

1. Регистрация, вход, обновление токена, выход и восстановление сессии после перезагрузки страницы.
2. Просмотр dashboard со списком портфелей с фильтрацией, сортировкой и пагинацией.
3. CRUD-операции с портфелями, активами и транзакциями владельцем портфеля.
4. Поведение ролей:
   - `user` работает только со своими сущностями;
   - `manager` видит все портфели, но не должен менять чужие данные;
   - `admin` управляет всеми портфелями и ролями пользователей.
5. Импорт публичного портфеля по ID с копированием активов и транзакций.
6. Загрузка файлов, выдача защищённой ссылки на скачивание, публичное скачивание по signed token и удаление файла.
7. Импорт данных с Bybit и отображение рыночных котировок.

## 2. Ключевые бизнес-правила и ограничения

1. Все `/api/v1/portfolios*` и `/api/v1/integrations*` требуют валидный access token.
2. Email при регистрации должен быть валиден, а повторная регистрация запрещена.
3. Каждый новый пользователь получает роль `user`.
4. При логине старые refresh-сессии пользователя инвалидируются.
5. Refresh token должен быть непросроченным, неотозванным и не должен использоваться повторно после ротации.
6. Только `admin` может менять роли пользователей и удалять чужие портфели.
7. `manager` может просматривать чужие портфели, но не должен выполнять операции записи над ними.
8. `user` может работать только со своими портфелями и вложенными сущностями.
9. Актив внутри одного портфеля должен иметь уникальный `symbol`.
10. Количество транзакции должно быть больше нуля, а для `buy/sell` обязателен `price_usd`.
11. Размер файла ограничен `5 MB`, а тип файла должен входить в whitelist.
12. Скачать файл можно только с валидным неистёкшим presigned token.
13. Импортировать можно только публичный портфель.
14. Отказы внешних интеграций должны возвращаться контролируемой ошибкой, а не 500 без контекста.

## 3. Области повышенного риска

1. Аутентификация и истечение сессии: refresh flow, очистка токенов, повторные запросы после 401.
2. Права доступа: разграничение `user/manager/admin`, защита маршрутов React и API.
3. Работа с файлами: валидация payload, генерация signed token, контроль доступа к содержимому файла.
4. Внешние интеграции:
   - Bybit ticker для расчёта метрик портфеля;
   - Bybit portfolio import с обработкой rate limit и upstream errors.
5. Согласованность данных в CRUD-сценариях: уникальность активов, видимость публичных портфелей, пагинация и фильтрация.
